# Meme Bot

Бот парсит Telegram-каналы, присылает найденные мемы тебе в личку на одобрение и публикует одобренные в твой канал по расписанию.

---

## ВАЖНО: архитектура

**Бот живёт на Railway.** Мак — только для редактирования кода.

- Railway → единственный запущенный инстанс бота
- Мак → редактируешь код, делаешь `git push` → Railway передеплоится сам
- **Никогда не запускай бота локально** — если локальный и Railway-инстанс работают одновременно, Telegram отдаёт обновления то одному то другому, кнопки перестают отвечать

### Если кнопки вдруг не работают

Причина почти всегда одна: **на маке случайно запущен второй инстанс бота**.

Проверить и убить:
```bash
pgrep -fa bot.py          # найти процесс
kill <PID>                # убить его
```

Убедиться что оба LaunchAgent выгружены (должны быть **выгружены навсегда**):
```bash
launchctl list | grep memebot   # должна быть пустая строка
```

Если что-то есть — выгрузить:
```bash
launchctl unload ~/Library/LaunchAgents/com.possum.memebot.plist
launchctl unload ~/Library/LaunchAgents/com.possum.memebot.healthcheck.plist
```

### Почему healthcheck.py нельзя использовать

`healthcheck.py` проверяет наличие `bot.pid` и если его нет — **запускает бота локально**. Так как бот на Railway не создаёт локальный `bot.pid`, каждое утро в 9:00 healthcheck поднимал второй инстанс, который конфликтовал с Railway → кнопки переставали работать.

`healthcheck.py` и оба LaunchAgent (`com.possum.memebot`, `com.possum.memebot.healthcheck`) должны оставаться **выгруженными**. Не загружать их снова.

## Что умеет

- Парсит **28+ источников** параллельно каждый час
- Фильтрует рекламу, ссылки, длинные подписи автоматически
- Дедуплицирует мемы по phash — одинаковые картинки не пройдут дважды
- Присылает мемы в личку с кнопками — одобрить, пропустить, добавить подпись или запостить сразу
- Публикует по расписанию **7–10 раз в день** с 9:00 до 22:00 МСК
- Уведомляет после каждой публикации с остатком очереди
- Сохраняет картинки в базе — мем опубликуется даже если CDN-ссылка протухла
- Поддерживает альбомы (несколько фото в одном посте)

## Команды

| Команда | Что делает |
|---------|------------|
| `/fetch` | Проверить каналы прямо сейчас и получить новые мемы |
| `/post` | Опубликовать следующий мем из очереди вручную |
| `/schedule` | Расписание публикаций на сегодня с обратным отсчётом |
| `/queue` | Сколько мемов ждут публикации |
| `/status` | Статистика базы (новые, одобренные, пропущенные, опубликованные) |
| `/skip` | Одобрить мем без подписи (когда бот ждёт текст) |
| `/clearsent` | Сбросить все непросмотренные мемы — чистый лист перед `/fetch` |
| `/clearqueue` | Удалить из очереди битые посты без картинки |
| `/showqueue` | Показать очередь (публикация, подпись, удаление) |
| `/listchannels` | Список каналов-источников |
| `/addchannel @username` | Добавить канал |
| `/removechannel @username` | Удалить канал |
| `/trustchannel @username` | Авто-одобрение постов из этого канала |
| `/help` | Показать все команды прямо в боте |

## Кнопки при одобрении

| Кнопка | Действие |
|--------|---------|
| ✅ | Добавить в очередь без подписи |
| 🚀 | Опубликовать прямо сейчас |
| ✍️ | Добавить в очередь с подписью |
| ❌ | Пропустить |

## Деплой (Railway)

### Текущая конфигурация

- Репозиторий: `nastyashanina88/memebot` на GitHub
- Railway подключён к этому репозиторию и деплоит автоматически при пуше в `main`
- Procfile: `worker: python bot.py`

### Переменные окружения в Railway

| Переменная | Описание |
|------------|---------|
| `BOT_TOKEN` | Токен от @BotFather |
| `MY_CHANNEL` | `@yslovnay` |
| `ADMIN_CHAT_ID` | Telegram ID для получения мемов (задаётся через `/start`) |
| `DATA_DIR` | `/data` — путь к Volume с базой данных |

### Volume

В Railway добавлен Volume с Mount Path `/data` — там хранится `memes.db`. Без него база сбрасывается при каждом редеплое.

### Как задеплоить изменение

```bash
git add .
git commit -m "описание изменений"
git push
```

Railway подхватит пуш и передеплоит автоматически (~1 минута).

## Настройки (bot.py)

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `SOURCE_CHANNELS` | 29 каналов | Откуда брать мемы |
| `POSTS_PER_DAY_MIN/MAX` | 7–10 | Сколько постов в день |
| `POST_START_HOUR` | 9 | С какого часа (МСК) |
| `POST_END_HOUR` | 22 | До какого часа (МСК) |
| `MAX_CAPTION_LEN` | 150 | Максимальная длина подписи |
| `FETCH_INTERVAL` | 3600 | Как часто проверять каналы (сек) |
| `FETCH_HOURS_BACK` | 24 | Брать посты за последние N часов |
| `MAX_SEND_PER_FETCH` | 25 | Максимум мемов на одобрение за один /fetch |
| `PHASH_THRESHOLD` | 10 | Порог похожести картинок (меньше = строже) |
