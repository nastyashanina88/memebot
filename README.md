# Meme Bot

Бот парсит Telegram-каналы, присылает найденные мемы тебе в личку на одобрение и публикует одобренные в твой канал по расписанию.

---

## ВАЖНО: архитектура

**Бот живёт на Railway.** Мак — только для редактирования кода.

- Railway → единственный запущенный инстанс бота
- Мак → редактируешь код, делаешь `git push` → Railway передеплоится сам
- **Никогда не запускай бота локально** — два инстанса одновременно = кнопки не работают

---

## Если что-то сломалось

### Кнопки не реагируют

**Причина 1: на маке запущен второй инстанс**
```bash
pgrep -fa bot.py    # если что-то нашлось:
kill <PID>
```

Убедись что LaunchAgents выгружены (навсегда):
```bash
launchctl list | grep memebot   # должно быть пусто
# если не пусто:
launchctl unload ~/Library/LaunchAgents/com.possum.memebot.plist
launchctl unload ~/Library/LaunchAgents/com.possum.memebot.healthcheck.plist
```

**Причина 2: база данных переполнена** (`database or disk is full` в логах Railway)
```
Написать боту: /vacuum
```
Это очистит бинарные данные старых постов и освободит место.

**Причина 3: бот упал на Railway**
- Зайти в Railway → worker → Deployments
- Если CRASHED — нажать Restart
- Если падает снова — смотреть Deploy Logs, скинуть ошибку

### Бот не публикует по расписанию

```
/queue    — проверить очередь
/schedule — проверить расписание
/fetch    — вручную загрузить новые мемы
```

## Что умеет

- Парсит **29 источников** параллельно каждый час
- Фильтрует рекламу, ссылки, длинные подписи
- Дедуплицирует по phash — одинаковые картинки не пройдут дважды
- Присылает мемы в личку с кнопками одобрения
- Публикует по расписанию **7–10 раз в день** с 9:00 до 22:00 МСК
- Поддерживает альбомы (несколько фото)
- Сохраняет картинки в базе — публикует даже если CDN протух

## Команды

| Команда | Что делает |
|---------|------------|
| `/fetch` | Проверить каналы и получить новые мемы |
| `/post` | Опубликовать следующий мем вручную |
| `/schedule` | Расписание на сегодня с обратным отсчётом |
| `/queue` | Сколько мемов в очереди |
| `/status` | Статистика базы |
| `/vacuum` | Очистить базу от старых бинарных данных (если диск забился) |
| `/skip` | Одобрить без подписи (когда бот ждёт текст) |
| `/clearsent` | Сбросить непросмотренные мемы |
| `/clearqueue` | Удалить битые посты из очереди |
| `/showqueue` | Показать очередь с кнопками управления |
| `/listchannels` | Список каналов-источников |
| `/addchannel @x` | Добавить канал |
| `/removechannel @x` | Удалить канал |
| `/trustchannel @x` | Авто-одобрение постов из канала |
| `/help` | Все команды в боте |

## Кнопки при одобрении

| Кнопка | Действие |
|--------|---------|
| ✅ | Добавить в очередь без подписи |
| 🚀 | Опубликовать прямо сейчас |
| ✍️ | Добавить с подписью |
| ❌ | Пропустить |

## Деплой (Railway)

### Конфигурация

- Репо: `nastyashanina88/memebot` на GitHub
- Автодеплой при пуше в `main`
- Procfile: `worker: python bot.py`
- Python: 3.11 (зафиксирован в `runtime.txt`)

### Переменные окружения в Railway

| Переменная | Значение |
|------------|---------|
| `BOT_TOKEN` | токен от @BotFather (без пробелов и переносов!) |
| `MY_CHANNEL` | `@yslovnay` |
| `ADMIN_CHAT_ID` | Telegram ID (задаётся через `/start`) |
| `DATA_DIR` | `/data` |

### Volume

Mount Path `/data` → там `memes.db`. Без Volume база сбрасывается при каждом редеплое.

### Деплой изменений

```bash
git add .
git commit -m "описание"
git push
# Railway передеплоится автоматически (~1 мин)
```

## Настройки (bot.py)

| Параметр | Значение | Описание |
|----------|---------|----------|
| `POSTS_PER_DAY_MIN/MAX` | 7–10 | Постов в день |
| `POST_START_HOUR` | 9 | С какого часа МСК |
| `POST_END_HOUR` | 22 | До какого часа МСК |
| `FETCH_INTERVAL` | 3600 | Проверка каналов (сек) |
| `FETCH_HOURS_BACK` | 24 | Брать посты за N часов |
| `MAX_SEND_PER_FETCH` | 25 | Мемов на одобрение за раз |
| `PHASH_THRESHOLD` | 10 | Похожесть картинок (меньше = строже) |
| `MAX_CAPTION_LEN` | 150 | Макс. длина подписи |
